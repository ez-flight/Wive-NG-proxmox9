# Wive-NG — виртуальный роутер (Proxmox)

## О проекте Wive-NG-HQ

**Wive-NG-HQ** (*Wireless Comprehensive Advanced Technology project*) — это линейка открытой прошивки для сетевого оборудования в рамках экосистемы **Wive-NG**: встраиваемая Linux-система для маршрутизаторов и точек доступа с упором на **беспроводные технологии**, расширяемый стек сервисов и «продвинутую» настройку сети без ограничений типичных заводских прошивок.

**Wive-NG** исторически развивается как проект альтернативной ОС для роутеров на разных платформах (в т.ч. SoC Realtek, Ralink/MediaTek и др.); для пользователей и интеграторов доступны сборки под конкретное железо, а также **виртуальная x86_64-сборка** — полноценный программный маршрутизатор в ВМ (VirtualBox, QEMU/KVM, **Proxmox VE**).

Типичные возможности семейства Wive-NG (зависят от платформы и версии сборки):

- маршрутизация, NAT, firewall, QoS;
- Wi‑Fi и проводные интерфейсы (в железных сборках), VPN (OpenVPN, WireGuard и др.), туннелирование;
- сервисы уровня сети: DHCP, DNS, UPnP, IGMP/мультикаст-прокси, мониторинг (в т.ч. Zabbix-агент в ряде образов), TR-069 (CWMP) и др.;
- веб-интерфейс администрирования и гибкая настройка под задачи SOHO/операторского доступа.

Сборка **x86_64** из этого каталога предназначена для запуска **виртуального роутера** в датацентре или на ПК: удобно для тестов, обучения и переноса конфигураций без отдельного физического устройства.

Официальные материалы и актуальные ветки проекта см. на [SourceForge: wive-ng](https://sourceforge.net/projects/wive-ng/) и сайте проекта **[wive-ng.ru](https://wive-ng.ru/)** (в т.ч. документация, форумы партнёров вроде Wi-CAT). Условия лицензирования и коммерческого использования компонентов — у правообладателей и в описании конкретной сборки.

Каталог ниже содержит конфигурацию **Oracle VirtualBox** и данные для переноса образа в **Proxmox VE**.

## Состав файлов

| Файл | Назначение |
|------|------------|
| `Wive.vbox` | Описание ВМ VirtualBox (память, CPU, сеть, ссылка на диск). |
| `drive.vmdk` | Дескриптор VMDK (`fullDevice`), указывает на **`bootfs.img`**. |
| `bootfs.img` | Реальный **raw**-образ диска (~7.5 ГиБ). **Обязателен для импорта в Proxmox.** Можно получить из архива (см. ниже). |

Импорт в Proxmox выполняется по **`bootfs.img`**, а не по `drive.vmdk`.

### Официальная сборка x86_64 (SourceForge)

Архив с образом диска и файлами для VirtualBox:

**[wive-x86_64-9.8.6-13.10.2025.tar.xz (download)](https://sourceforge.net/projects/wive-ng/files/wive-x86-64/wive-x86_64-9.8.6-13.10.2025.tar.xz/download)**

Скрипт `deploy-wive-proxmox.sh` по умолчанию **скачивает этот архив**, распаковывает его и берёт из него **`bootfs.img`** (если локального `bootfs.img` ещё нет).

## Требования

- **Proxmox VE** 7.x / 8.x с утилитами `qm`, `pvesm`, `tar`, `xz` (для `.tar.xz`), `curl` или `wget`.
- Хранилище виртуальных дисков (например `local-lvm`).
- Свободный **VMID**.
- Достаточно места в thin pool / LVM под ~8 ГиБ.

## Важно (после миграции с VirtualBox)

1. **Диск** — подключать как **SATA** (`sata0`), не как virtio-scsi: иначе прошивка может зависнуть на сообщении вроде *wait ready device with system config parts before boot*.
2. **Сеть** — для Wive часто нужны имена **`eth0` / `eth1`**. В Proxmox задайте модель **`e1000`** (или при необходимости `rtl8139`), а не virtio.

## Быстрый старт

1. Скопируйте каталог **Wive** на узел **Proxmox** (или клонируйте репозиторий, если он у вас в Git).
2. Запуск от **root** на узле Proxmox (по умолчанию скачивается архив SourceForge и извлекается `bootfs.img`):

```bash
cd "/path/to/Wive"
chmod +x deploy-wive-proxmox.sh
sudo ./deploy-wive-proxmox.sh
```

3. Если **`bootfs.img`** уже лежит рядом со скриптом — скачивание **пропускается**. Чтобы принудительно скачать архив заново: `export FORCE_DOWNLOAD=1`.

4. Параметры окружения:

```bash
export VMID=123
export STORAGE=local-lvm
export BRIDGE_WAN=vmbr1
export BRIDGE_LAN=vmbr2
export RAM=1024
export CORES=4
# другой архив или зеркало:
# export WIVE_ARCHIVE_URL="https://sourceforge.net/projects/wive-ng/files/..."
# прямой URL только на bootfs.img (приоритет над архивом):
# export WIVE_IMAGE_URL="https://..."
sudo -E ./deploy-wive-proxmox.sh
```

| Переменная | Назначение |
|------------|------------|
| **`WIVE_ARCHIVE_URL`** | Ссылка на **`.tar.xz`** (по умолчанию — сборка 9.8.6 от 13.10.2025). |
| **`WIVE_IMAGE_URL`** | Прямая ссылка на **`bootfs.img`** (если задана — архив не качается). |
| **`SKIP_DOWNLOAD=1`** | Не качать; нужен готовый **`bootfs.img`** в каталоге. |
| **`KEEP_ARCHIVE=1`** | После распаковки **не удалять** скачанный `.tar.xz`. |

## После запуска ВМ

- Проверьте интерфейсы: `ip link` в консоли гостя.
- Настройте сеть и веб-интерфейс в соответствии с документацией вашей сборки Wive/Wive-NG.
